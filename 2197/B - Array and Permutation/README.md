Array and Permutation</div><div class="time-limit"><div class="property-title">time limit per test</div>1.5 seconds</div><div class="memory-limit"><div class="property-title">memory limit per test</div>256 megabytes</div><div class="input-file input-standard"><div class="property-title">input</div>standard input</div><div class="output-file output-standard"><div class="property-title">output</div>standard output</div></div><div><p> </p><p>Given a permutation $$$p$$$ of length $$$n$$$$$$^{\text{∗}}$$$ and an array $$$a$$$ of length $$$n$$$.</p><p>We call the permutation $$$p$$$ <span class="tex-font-style-it">generating</span> for the array $$$a$$$ if the array $$$a$$$ can be obtained from the permutation $$$p$$$ by applying some number of operations (possibly zero) of the following type:</p><p>Choose an index $$$i$$$ ($$$1 \le i  \lt  n$$$) and perform one of two replacements: </p><ul> <li> $$$p_{i} := p_{i + 1}$$$; </li><li> $$$p_{i + 1} := p_{i}$$$. </li></ul><p>In other words, in one operation, you can choose two adjacent elements of the array and copy the value of one into the other.</p><p>You are required to report whether the permutation $$$p$$$ is <span class="tex-font-style-it">generating</span> for the array $$$a$$$.</p><div class="statement-footnote"><p>$$$^{\text{∗}}$$$A permutation of length $$$n$$$ is an array consisting of $$$n$$$ distinct integers from $$$1$$$ to $$$n$$$ in arbitrary order. For example, $$$[2,3,1,5,4]$$$ is a permutation, but $$$[1,2,2]$$$ is not a permutation ($$$2$$$ appears twice in the array), and $$$[1,3,4]$$$ is also not a permutation ($$$n=3$$$ but there is $$$4$$$ in the array). </p></div></div><div class="input-specification"><div class="section-title">Input</div><p>Each test contains multiple test cases. The first line contains the number of test cases $$$t$$$ ($$$1 \le t \le 10^4$$$). The description of the test cases follows. </p><p>The first line of each test case contains a single integer $$$n$$$ ($$$2 \le n \le 2 \cdot 10^{5}$$$) — the length of the array and the permutation.</p><p>The second line of each test case contains $$$n$$$ integers $$$p_1, p_2, \ldots, p_n$$$ ($$$1 \le p_{i} \le n$$$) — the elements of the permutation.</p><p>The third line of each test case contains $$$n$$$ integers $$$a_1, a_2, \ldots, a_n$$$ ($$$1 \le a_{i} \le n$$$) — the elements of the array.</p><p>It is guaranteed that the sum of $$$n$$$ across all test cases does not exceed $$$2 \cdot 10^{5}$$$.</p></div><div class="output-specification"><div class="section-title">Output</div><p>For each test case, output "<span class="tex-font-style-tt">YES</span>" if the permutation $$$p$$$ is <span class="tex-font-style-it">generating</span> for the array $$$a$$$, otherwise output "<span class="tex-font-style-tt">NO</span>".</p><p>You may output each letter in any case (lowercase or uppercase).
