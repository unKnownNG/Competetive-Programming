<div class="header"><div class="title">A. Course Wishes</div><div class="time-limit"><div class="property-title">time limit per test</div>1 second</div><div class="memory-limit"><div class="property-title">memory limit per test</div>256 megabytes</div><div class="input-file input-standard"><div class="property-title">input</div>standard input</div><div class="output-file output-standard"><div class="property-title">output</div>standard output</div></div><div><p>qwqkawaii is registering for $$$n$$$ ($$$n\le 50$$$) courses. In the registration system, he can submit a <span class="tex-font-style-it">course wish</span> for each course, which indicates his priority for taking that course.</p><p>Course wishes are divided into $$$k+1$$$ ($$$k\le 20$$$) priority levels, where level $$$1$$$ is the highest priority and level $$$k+1$$$ is the lowest.</p><p>The first $$$k$$$ wish levels have capacity limits: for each $$$1 \le i \le k$$$, at most $$$a_i$$$ courses can be assigned wish level $$$i$$$. Note that wish level $$$k+1$$$ has no capacity limit.</p><p>Initially, the $$$i$$$-th course has wish level $$$b_i$$$, and it is guaranteed that this initial assignment satisfies all capacity limits. Now qwqkawaii wants to adjust all his courses to wish level $$$k + 1$$$. To achieve this, he can apply the following operation at most $$$1000$$$ times:</p><ul> <li> Select a course $$$i$$$ ($$$1\le i\le n$$$), then increase $$$b_i$$$ by $$$1$$$. </li></ul><p>Note that:</p><ul> <li> A course at level $$$k+1$$$ cannot be selected; </li><li> After every single operation, all capacity limits <span class="tex-font-style-bf">must</span> still be satisfied. </li></ul><p>Your task is to construct a valid adjustment sequence with at most $$$1000$$$ operations, or report that it is impossible.</p></div><div class="input-specification"><div class="section-title">Input</div><p>Each test contains multiple test cases. The first line contains the number of test cases $$$t$$$ ($$$1 \le t \le 50$$$). The description of the test cases follows. </p><p>The first line of each test case contains two integers $$$n$$$ and $$$k$$$ ($$$1 \le n \le 50$$$, $$$1 \le k \le 20$$$) — the number of courses and the number of priority levels (excluding the lowest priority level).</p><p>The second line contains $$$k$$$ integers $$$a_1, a_2, \ldots, a_k$$$ ($$$1 \le a_i \le n$$$) — the capacity limits of the first $$$k$$$ wish levels.</p><p>The third line contains $$$n$$$ integers $$$b_1, b_2, \ldots, b_n$$$ ($$$1 \le b_i \le k+1$$$) — the initial wish levels of the courses. </p><p>It is guaranteed that the initial assignment satisfies all capacity limits.</p></div><div class="output-specification"><div class="section-title">Output</div><p>For each test case, if it is impossible to reach the target state, print a single integer $$$-1$$$.</p><p>Otherwise, print the number of operations $$$m$$$ ($$$0 \le m \le 1000$$$) on the first line of output.</p><p>Then print one line with $$$m$$$ integers $$$u_1, u_2, \ldots, u_m$$$ ($$$1 \le u_i \le n$$$), denoting that in the $$$i$$$-th operation, you increase the wish level $$$b_{u_i}$$$ of course $$$u_i$$$ by $$$1$$$.</p></div><div
