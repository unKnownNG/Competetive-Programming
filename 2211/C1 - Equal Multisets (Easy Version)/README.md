The difference between the versions is that in this version, the array $$$a$$$ is guaranteed to be a permutation. You can hack only if you solved all versions of this problem.</span> </p><p>Given an array $$$a$$$ of size $$$n$$$, and a parameter $$$k$$$, an array $$$b$$$ is called <span class="tex-font-style-it">cool</span> if the following conditions hold:</p><ul> <li> For each $$$i$$$ from $$$k$$$ to $$$n$$$, the array $$$[a_{i-k+1},a_{i-k+2},\ldots,a_i]$$$ is a rearrangement of $$$[b_{i-k+1},b_{i-k+2},\ldots,b_i]$$$. </li></ul><p>You are given two arrays $$$a$$$ and $$$b$$$ of size $$$n$$$, and an integer $$$k$$$. The array $$$a$$$ <span class="tex-font-style-bf">is guaranteed to be a permutation</span>$$$^{\text{∗}}$$$. The array $$$b$$$ only contains integers from $$$1$$$ to $$$n$$$, and $$$-1$$$.</p><p>Determine if it is possible to replace all $$$-1$$$ in $$$b$$$ with an integer from $$$1$$$ to $$$n$$$, such that $$$b$$$ is <span class="tex-font-style-it">cool</span> with respect to $$$k$$$.</p><div class="statement-footnote"><p>$$$^{\text{∗}}$$$A permutation of length $$$n$$$ is an array consisting of $$$n$$$ distinct integers from $$$1$$$ to $$$n$$$ in arbitrary order. For example, $$$[2,3,1,5,4]$$$ is a permutation, but $$$[1,2,2]$$$ is not a permutation ($$$2$$$ appears twice in the array), and $$$[1,3,4]$$$ is also not a permutation ($$$n=3$$$ but there is $$$4$$$ in the array). </p></div></div><div class="input-specification"><div class="section-title">Input</div><p>Each test contains multiple test cases.
